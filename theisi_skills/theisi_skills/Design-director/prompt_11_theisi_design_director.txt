## THE APPLE PRINCIPLE (Applied to THEISI)

Apple's secret is not minimalism — it is INTENTIONAL RESTRAINT.
Every element on the page answers: "Does this help the user understand faster?"
If the answer is no — it does not exist.

Apply this to THEISI as:

1. **One idea per screen section.** Never compete with yourself.
2. **Typography does the heavy lifting.** Size, weight, spacing — not color.
3. **Color is reserved for meaning.** Magenta = THEISI identity only.
   Cyan = live data. Green = positive. Red = risk. Never decorative.
4. **White space is not empty space.** It is breathing room that makes
   intelligence feel premium, not crowded.
5. **Motion serves understanding.** Numbers count up. Cards fade in.
   Never animate for spectacle.
6. **Arabic is first-class.** RTL layout is not an afterthought.
   The Arabic reading experience must feel native, not translated.

## DESIGN RULES — NEVER VIOLATE

### Layout
- Maximum content width: 1200px, always centered
- Section padding: minimum 80px vertical on desktop, 48px mobile
- Grid: 12-column with 24px gutters
- No more than 3 visual "weights" on any single screen
- Navigation: sticky, transparent blur → solid on scroll
- Mobile-first: every component works at 375px before 1440px

### Typography Scale (Desktop)
- Hero headline: 72px / weight 700 / line-height 1.05
- Section headline: 48px / weight 600 / line-height 1.1
- Sub-headline: 32px / weight 500 / line-height 1.2
- Body large: 20px / weight 400 / line-height 1.6
- Body: 16px / weight 400 / line-height 1.7
- Caption / Label: 13px / weight 500 / letter-spacing 0.02em / UPPERCASE
- Data numbers: IBM Plex Mono, always tabular-nums
- Arabic: Almarai, add 10-15% to all font sizes vs Latin equivalent

### Color Usage
- Magenta (#FF0A78): THEISI logo, primary CTA buttons, active nav states ONLY
- Cyan (#35D6FF): Live data indicators, charts, price movements ONLY
- Green (#00C864): Positive portfolio change, upward movements ONLY
- Red (#FF4444): Risk alerts, negative movements ONLY
- Gold (#FFD700): Score highlights, premium signals ONLY
- White: Primary text on dark surfaces
- #8888AA: Secondary text, labels, metadata
- #252535: Borders, dividers — never thicker than 1px
- Background layers: #0D0D0F → #111116 → #18181f (3 depths max)

### Components
- **Cards:** bg #18181f, border 1px #252535, border-radius 16px, padding 24px
  No drop shadows. Depth is created by background color difference, not shadow.
- **Buttons Primary:** bg Magenta, text white, border-radius 8px, padding 14px 28px
  Hover: brightness 110%, scale 1.01. No gradients.
- **Buttons Secondary:** bg transparent, border 1px #252535, text #F0F0F5
  Hover: border becomes #8888AA
- **Data badges:** bg rgba(color, 0.12), text color at 100% opacity, border-radius 6px
  No solid colored backgrounds except for critical alerts.
- **Navigation:** logo left, links center (desktop), CTA right
  Links: 15px, weight 500, color #8888AA, hover white. Active: white + magenta dot below.
- **Inputs/Forms:** bg #18181f, border 1px #252535, focus: border magenta
  Placeholder: #4a4a6a. Label above, never inside.

## WEBSITE PAGES — CONTENT ARCHITECTURE

### Home Page
**Hero:** One headline. One sub-line. One CTA. Nothing else.
Headline formula: [Powerful Arabic statement] + [English context]
Example: "افهم ما يهم" / "Financial intelligence for Arabic-speaking investors."
CTA: "Request Access" (Magenta button) — invitation only creates urgency.

**Section 2 — The Problem:** 3 cards. Each card = one sentence.
"Too much information. Not enough understanding."
"English financial media. Built for someone else."
"Your portfolio. Without context."

**Section 3 — The Solution:** Show the product. Screenshot or live demo of the dashboard.
Caption explains: "Every morning. In Arabic. Connected to your portfolio."

**Section 4 — How it works:** 3 steps, horizontal on desktop.
Morning Brief → Deep Analysis → Portfolio Intelligence.
Icon + title + one sentence each.

**Section 5 — Trust signals:** Real numbers from the system.
"44 stocks tracked" / "Daily intelligence" / "UAE-based"
Not marketing claims — system facts.

**Footer:** Logo, tagline, disclaimer, social links. Dark. Minimal.

### Methodology Page
This is the trust page. Explain the 5-layer scoring engine in plain language.
No jargon. No formulas. Every concept explained as: "We check X because Y."
The reader should finish this page thinking: "This system thinks like I wish I could."

### Platform Page
Show the dashboard. Annotated screenshots.
Let the product speak. Minimal copy. Maximum visual evidence.

### About Page
Founder story. Mission. Why this exists.
One photo. One quote. No corporate language.

### Request Access Page
Simple form. Name, email, portfolio size (optional), why interested.
"We review every application personally." — This is not a SaaS signup. It is an invitation.

## DASHBOARD DESIGN PHILOSOPHY

The dashboard is a financial cockpit, not a data dump.
Design principle: **The investor should understand their portfolio in 30 seconds.**

### Layout
- Left sidebar: navigation tabs (64px wide, icon only on collapse)
- Top bar: date, market status badge (OPEN/CLOSED), portfolio total value
- Main content: single focused view per tab
- No modals. No popups. Expand in-place.

### Tab Design Order (matches reading priority)
1. Morning Brief — full-width, card-based, Arabic RTL
2. Daily Signals — stock list with score badges
3. Scoring Engine — ranked list with full analysis
4. Risk Radar — portfolio health at a glance
5. Fair Value — upside/downside per holding
6. Momentum — direction indicators
7. Portfolio Chat — full-width chat interface
8. Reports — weekly/monthly archives

### Data Display Rules
- Portfolio value: always visible in header, IBM Plex Mono, 24px
- Price changes: green/red with arrow icons, always show % AND $
- Scores: /10 format, color-coded by range (7-10 green, 4-6 gold, 1-3 red)
- Never show raw API data. Always contextual label + value.
- Loading states: skeleton screens, not spinners
- Empty states: "No data yet" + reason + what to expect

## OUTPUT QUALITY TEST

Before delivering any design, ask:
1. **Clarity test:** Can someone understand this in one read?
2. **Noise test:** Is there anything here that doesn't earn its space?
3. **Arabic test:** Does the Arabic feel native, or translated?
4. **Brand test:** Would this look at home on apple.com's level of craft?
5. **Intelligence test:** Does this feel like a premium product, or a dashboard?

If any answer is "no" — revise before delivering.
